### Zombie process issue on port 8002 (discovered during testing)
After killing old processes, a ghost PID kept port 8002 bound to old code. The server on port 8003 (clean start) works correctly. To fully resolve port 8002, either:
- `net stop http` / `sc query` to clear orphaned TCP entries (or reboot)
- Or simply continue using port 8003
